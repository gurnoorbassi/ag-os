# Dashboard Action Queue v1

Dashboard Action Queue v1 is a read-only dashboard read model for the owner's next decisions, blocked actions, approval package templates, stale approval warnings, and safe next milestones.

It does not execute commands, approve work, call connectors, store credentials, post content, schedule content, deploy, or change source-of-truth records.

## Source Records

The queue is generated by `scripts/build-dashboard.mjs` from existing AG OS records:

- `.codex/approvals/`
- `.codex/client-management/`
- `.codex/connectors/`
- `.codex/critiques/`
- Netlify staging connector proof records when present
- existing dashboard and policy docs

Missing data must render as `Not recorded`, `blocked`, `pending_owner_input`, or another explicit non-executing state. The dashboard must not invent approvals, handles, credentials, connector status, or metrics.

## Queue Sections

The read model surfaces:

- owner decisions needed
- blocked actions
- approval packages ready
- stale approvals
- next safe command
- manual posting availability
- OAuth blocked reason
- credential-store missing reason
- safe next milestones
- one read-only batched approval review containing owner decisions, ready approval packages, and active standing approval usage/expiry state

## Safety Rules

Dashboard Action Queue v1 is visibility only.

- Approval package templates do not grant permission.
- Critiques do not grant approval.
- Candidate lessons do not become accepted truth.
- Browser state is not source of truth.
- No dashboard element may execute a live action.
- No button, form, input, localStorage, fetch, or connector call is allowed in this dashboard version.
- The batched approval review cannot create, modify, or grant approvals. It only groups source-controlled records for owner review.

Any future authenticated dashboard action requires a separate owner-approved authenticated dashboard actions plan and approval lock.
