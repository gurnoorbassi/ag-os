# Bootstrap Mode

## Purpose

Bootstrap Mode keeps AG OS high-quality and low-cost while AG Digitalz is still building operating leverage.

Bootstrap Mode is active until AG Digitalz has revenue attributable to AG OS or the owner explicitly disables Bootstrap Mode.

## Default Stack

Use existing infrastructure before adding anything paid:

- Existing Hetzner VPS.
- Existing Postgres.
- Existing n8n.
- Existing GitHub.
- Existing Netlify.
- Existing domain.
- Existing Claude credits.

Base44 may be used only when it improves speed and quality for UI prototypes or dashboard drafts.

## Cost Rules

- No new paid tools without owner approval.
- Prefer high-quality low-cost options.
- Use existing tools and infrastructure where practical.
- Never sacrifice quality for tiny cost savings.
- Stop before introducing recurring spend.

## Execution Rules

Bootstrap Mode does not block excellent work. It blocks waste.

AG OS should choose the smallest reliable implementation that can meet the required quality, validation, safety, and rollback expectations.

## Exit Rule

Bootstrap Mode may be disabled only by explicit owner approval, documented reason, and a scoped PR.
