# AG OS Documentation Map

## Purpose

This map prevents supporting documents from competing with the active Constitution or repeating the same rules with different wording. It changes no authority, approval, or safety rule.

## Canonical Sources

| Subject | Canonical source | Supporting implementation detail |
| --- | --- | --- |
| Operating authority and constitutional rules | `docs/ag-os-constitution-v1.md` | `docs/action-matrix.md` and the active approval records |
| Completion vision and horizon status | `docs/ag-os-vision-roadmap-v1.md` | `docs/build-mode-roadmap.md` |
| Mandatory boot policy | `docs/boot-sequence.md` | `docs/boot-sequence-runner.md` and `scripts/boot-check.mjs` |
| Validation coverage and limitations | `docs/validation-limits.md` | `scripts/validate-foundation.mjs` and validator tests |
| Lesson lifecycle | `docs/unified-memory-learning-os.md` | `docs/lesson-promotion-policy.md` and `docs/lesson-promotion-workflow.md` |
| Skills lifecycle | `docs/skills-library.md` | Active records under `.codex/skills/` |
| Production readiness | `docs/production-management-readiness-package.md` | Project-specific readiness records under `.codex/production/` |
| Private coordinator operation | `docs/live-operator-runtime.md` | `docs/ag-os-coordinator-deployment-runbook.md` |

## Historical Documents

`docs/constitution-readiness.md` is the retained pre-activation checklist and activation history. It is not a current authority source. The active Constitution records the current governing state.

## Conflict Rule

If supporting documentation conflicts with the Constitution, the Constitution wins. If two supporting documents conflict, use the canonical source in this map and open a scoped documentation correction. Documentation never grants permission for a gated action.
