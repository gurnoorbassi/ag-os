# Dashboard Read Model

The dashboard read model is generated by `scripts/build-dashboard.mjs` into `dashboard/dashboard-data.js`.

The read model is deterministic and source-controlled. It reads AG OS records from the local repository and does not call live services.

## Data Sources

The generator may read:

- `docs/ag-os-constitution-v1.md`
- `.codex/projects/registry.json`
- `.codex/projects/*.json`
- `.codex/connectors/registry.json`
- `.codex/connectors/connector-exec-*.json`
- `.codex/commands/registry.json`
- `.codex/capabilities/registry.json`
- `.codex/costs/budget.json`
- `.codex/costs/cost-ledger-*.json`
- `.codex/approvals/*.json`
- `.codex/approvals/archive/*.json`
- `.codex/quality/policy.json`
- `.codex/security/policy.json`
- `.codex/watchdog/policy.json`
- `.codex/memory/policy.json`
- `.codex/quality-scores/*.json`
- `.codex/critiques/*.json`
- `.codex/memory/lessons/candidates/*.json`
- `.codex/skills/*.json`
- `.codex/client-management/**`
- `docs/first-client-intake-needed.md`

## Live-Service Boundary

The generator must not call:

- GitHub APIs
- Netlify APIs
- n8n APIs
- social APIs
- analytics APIs
- production services
- customer systems

Connector status must come from source-controlled connector registry and connector execution records only.

## Unknown and Missing Data

If a value is not recorded in AG OS, the read model must show:

- `Not recorded`
- `unknown`
- `blocked`
- `0`

It must not infer missing URLs, client names, active workflows, social accounts, analytics state, production status, or credentials.

For first-client readiness, the read model may parse `docs/first-client-intake-needed.md` for `REQUIRED_*` placeholders. If placeholders remain, the dashboard must show `intake_needed`, zero active client records, and the exact missing field count instead of inventing client data.

## Permission Boundary

The dashboard is not an execution surface. It may show approval records and safe-merge rules, but it cannot:

- approve actions
- create approval locks
- merge pull requests
- deploy
- connect services
- activate workflows
- post or schedule content
- write client records
- promote lesson candidates

Skills shown in the dashboard are advisory records. `skillsGrantPermission` must remain `false`.

Owner-attention cards are read-only summaries. They do not grant owner approval, create approval locks, or authorize live action.

## Staleness Rule

`dashboard/dashboard-data.js` is generated output. Any source record or generator change that affects the read model requires:

```powershell
npm.cmd run dashboard:build
```

`npm.cmd run validate` runs the dashboard checker and must fail when `dashboard/dashboard-data.js` is stale.
