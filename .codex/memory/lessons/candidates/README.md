# Lesson Candidates

Candidates are proposed lessons awaiting review. They are not truth and must not be treated as current operating knowledge. Every record must have `status: "candidate"` and validate against `schemas/lesson.schema.json`.

Promotion moves a record to `.codex/memory/accepted/` with `status: "accepted"` through owner approval and a reviewed PR, with proof recorded in the `promotion` block. Rejected candidates are preserved in `.codex/memory/rejected/` when the rejection itself is useful.
